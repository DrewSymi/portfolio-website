IAM Automation & Reporting Dashboard

PowerShell-driven dashboard for identity risk visibility — failed logins, account lockouts, privileged activity, and MFA trends.

PowerShell Microsoft Graph Reporting Monitoring Governance

Problem Statement

Identity data lives in silos. Sign-in logs in one portal. MFA status in another. Privileged group membership buried in Active Directory. When something goes wrong, teams waste hours stitching together information from multiple sources.

This project demonstrates how to automate data collection and create a unified view of identity risk indicators — giving security teams actionable visibility instead of data sprawl.

Why This Matters in IAM

You can't protect what you can't see. Identity risk visibility enables:

  • Early Detection: Spot brute force attempts before they succeed
  • Compliance Evidence: Demonstrate MFA coverage to auditors
  • Hygiene Tracking: Identify stale accounts before they become attack vectors
  • Privileged Oversight: Know who has elevated access at any moment
  • Trend Analysis: Understand if security posture is improving or degrading

Sample Automation

# Get failed sign-ins from last 24 hours
$StartDate = (Get-Date).AddDays(-1).ToString("yyyy-MM-ddTHH:mm:ssZ")

$FailedSignIns = Get-MgAuditLogSignIn -Filter `
    "status/errorCode ne 0 and createdDateTime ge $StartDate" `
    -Top 1000

# Group by failure reason
$FailureAnalysis = $FailedSignIns | Group-Object {
    $_.Status.FailureReason
} | Select-Object Name, Count | Sort-Object Count -Descending

# Identify potential brute force targets
$TargetedAccounts = $FailedSignIns | Group-Object UserPrincipalName |
    Where-Object { $_.Count -gt 10 } |
    Select-Object Name, Count

Write-Host "Accounts with 10+ failed attempts:"
$TargetedAccounts | Format-Table -AutoSize

Risk Considerations

  • Alert Fatigue: Too many alerts means important ones get ignored
  • False Positives: Legitimate travel, VPN changes can trigger alerts
  • Data Retention: Log storage has compliance and cost implications
  • Access to Data: Dashboard access itself needs governance
  • Baseline Drift: "Normal" changes over time — thresholds need tuning

Lessons Learned

  • Executive dashboards need simplicity; security teams need drill-down detail
  • Trend data is often more valuable than point-in-time snapshots
  • Alert fatigue is real — focus on actionable, high-confidence signals
  • Defining "anomalous" requires understanding normal business patterns first
  • Automation reduces manual work but requires ongoing maintenance
